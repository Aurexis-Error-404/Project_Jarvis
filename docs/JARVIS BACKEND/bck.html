Python Async — The Only Mental Model You Need

20 minutes · Understand this first or everything else is confusing
// The Core Mental Model
Async Python is a waiting room. When your code is waiting for something — a network response, a file read, Claude's API to respond — instead of sitting there doing nothing, it goes back to the waiting room and lets other tasks run. When the thing it was waiting for is ready, it picks up from where it left off. You never have two things running at the same time — you have one thing running while others wait efficiently.

Why JARVIS needs async

JARVIS does several things that involve waiting: calling Claude API (1–3 seconds), scraping web pages (2–5 seconds), running the file watcher in the background. Without async, one slow API call would freeze Jarvis completely — no responses, no file watching, nothing. With async, while Claude is thinking, the file watcher still runs.

pythonsync vs async — the difference
# SYNC — freezes everything while waiting
def handle_query(query):
    result = call_claude(query)   # freezes for 2 seconds
    return result                 # nothing else can happen during those 2 seconds

# ASYNC — gives up control while waiting, lets others run
async def handle_query(query):
    result = await call_claude(query)  # "await" = go back to waiting room
    return result                       # file watcher kept running during those 2 seconds

# THE RULE: if a function does anything that waits (API call, file read,
# sleep, database query) → make it async + add await before the call

The three things you need to know about async in JARVIS

1
async def = this function can wait without blocking
Any function that calls an API, reads a file, or calls another async function must be declared async def. In JARVIS: your WebSocket handler, your Claude client, your Playwright scraper — all async.
2
await = call this async function and wait for it to finish
You can only use await inside an async def function. If you forget await, you get a coroutine object instead of the result — your code doesn't crash but returns nonsense.
3
asyncio.create_task() = run something in the background
For the file watcher — you don't await it, you fire it and forget. asyncio.create_task(file_watcher.start()) starts the watcher running in the background while your main WebSocket loop continues.
pythonasync patterns you will use in JARVIS
import asyncio
import httpx  # async HTTP client — use this instead of requests

# Pattern 1: calling Claude API (async HTTP call)
async def call_claude(messages: list) -> str:
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "https://api.anthropic.com/v1/messages",
            headers={"x-api-key": API_KEY},
            json={...}
        )
    return response.json()

# Pattern 2: WebSocket handler (called by FastAPI for each connection)
async def websocket_handler(websocket):
    while True:
        data = await websocket.receive_json()    # waits for message
        result = await call_claude(data["query"]) # waits for Claude
        await websocket.send_json(result)          # waits for send

# Pattern 3: background task (file watcher runs forever in background)
async def startup():
    asyncio.create_task(file_watcher_loop())  # fire and forget
    # execution continues immediately — watcher runs in background

# Pattern 4: running sync code without blocking (gitpython is sync)
import asyncio
loop = asyncio.get_event_loop()
result = await loop.run_in_executor(None, sync_git_function, args)
# run_in_executor runs sync functions in a thread pool — doesn't block
⚠
The one async mistake that breaks everything: Using requests library inside an async function. requests is synchronous — it blocks the entire event loop. Always use httpx (async HTTP) or the anthropic SDK (which handles this for you). Same rule: time.sleep() blocks — use await asyncio.sleep() instead.

FastAPI + WebSocket Server

25 minutes · The backbone of the backend
// The Core Mental Model
FastAPI is a web framework that maps URLs to Python functions. When something connects to ws://localhost:8765/ws, FastAPI calls your WebSocket handler function. When Electron sends a message, your function receives it. When your function sends a response, Electron receives it. That's the entire model.
pythonbackend/main.py — complete starter file
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.middleware.cors import CORSMiddleware
import asyncio, json, uvicorn

app = FastAPI()

# Allow Electron to connect (CORS — otherwise browser blocks it)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)

# Health check — Integration Lead uses this to verify server is running
@app.get("/health")
async def health():
    return {"status": "ok"}

# THE MAIN ENDPOINT — Electron connects here
@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()  # must accept the connection first
    print("Electron connected")

    try:
        while True:
            # Wait for a message from Electron
            data = await websocket.receive_json()
            event = data.get("event")

            if event == "user_query":
                await handle_user_query(websocket, data)

            elif event == "set_mode":
                await handle_mode_change(websocket, data)

            elif event == "session_end":
                await handle_session_end(websocket)

    except WebSocketDisconnect:
        print("Electron disconnected")  # normal — user closed the app
    except Exception as e:
        print(f"Error: {e}")
        await websocket.send_json({
            "event": "error",
            "message": str(e),
            "recoverable": True
        })

# Startup: launch background tasks
@app.on_event("startup")
async def startup_event():
    asyncio.create_task(start_file_watcher())  # proactive engine
    print("JARVIS backend started")

# Helper: send status update to Electron
async def send_status(ws: WebSocket, message: str):
    await ws.send_json({"event": "status_update", "message": message})

# Run the server
if __name__ == "__main__":
    uvicorn.run("main:app", host="0.0.0.0", port=8765, reload=True)

WebSocket Protocol for JARVIS

15 minutes · The contract between you and Frontend

Every message between Electron and Python is a JSON object with an event field. This is the complete protocol — you implement the server side, Frontend implements the client side. Both sides use the same event names exactly.

Direction Event Key Fields When
Electron → Python user_query query, mode, timestamp User presses Enter in overlay
Electron → Python set_mode mode: "cloud"|"local" User toggles secure/cloud switch
Electron → Python session_end — App window closes
Python → Electron jarvis_reply text, tool_calls_made, timestamp Claude finishes responding
Python → Electron context_surface bullets[], triggered_by, confidence Proactive engine fires
Python → Electron tool_call_status tool, status: "started"|"completed" Before and after each tool call
Python → Electron status_update message, mode Thinking, searching, etc.
Python → Electron report_generated file_path, title HTML report saved to disk
Python → Electron error message, recoverable Something broke
pythonhandle_user_query — the main handler function
async def handle_user_query(ws: WebSocket, data: dict):
    query = data["query"]
    mode = data.get("mode", "local")

    # Tell Electron we're thinking
    await send_status(ws, "Thinking...")

    try:
        # Call Claude with the tool loop
        response_text = await claude_client.run(query, mode, ws)

        # Send the final response
        await ws.send_json({
            "event": "jarvis_reply",
            "text": response_text,
            "timestamp": datetime_now()
        })

    except Exception as e:
        await ws.send_json({
            "event": "error",
            "message": str(e),
            "recoverable": True
        })

Codebase Reader Tool

20 minutes · Tool #1 — Claude reads your project files
// The Core Mental Model
This tool walks your project directory with pathlib and returns the file tree plus contents of relevant files. Claude calls it when it needs to see actual code to answer a question. You control what gets included — too much and Claude's context fills up, too little and answers are generic.
pythonbackend/tools/codebase_reader.py
from pathlib import Path
import json

# File extensions we care about
CODE_EXTENSIONS = {'.py', '.js', '.jsx', '.ts', '.tsx', '.json', '.md', '.txt'}

# Folders we always skip
SKIP_DIRS = {'node_modules', '.git', '.venv', 'venv', '__pycache__',
             '.next', 'dist', 'build', '.mypy_cache'}

def read_codebase(path: str = ".", depth: int = 2) -> dict:
    """
    Claude calls this to understand the project structure.
    Returns: file tree + content of key files
    Never raises — always returns something useful.
    """
    try:
        root = Path(path).resolve()
        if not root.exists():
            return {"error": f"Path {path} does not exist"}

        tree = []
        files_content = {}
        file_count = 0

        for item in _walk(root, root, depth, 0):
            tree.append(item["path"])

            # Read content for code files (up to 50 files, 200 lines each)
            if item["is_file"] and file_count < 50:
                content = _read_file(root / item["path"])
                if content:
                    files_content[item["path"]] = content
                    file_count += 1

        return {
            "root": str(root),
            "file_tree": tree,
            "file_count": file_count,
            "files": files_content
        }

    except Exception as e:
        return {"error": str(e), "file_tree": [], "files": {}}

def _walk(root, current, max_depth, current_depth):
    if current_depth > max_depth:
        return
    try:
        for item in sorted(current.iterdir()):
            if item.name.startswith(".") or item.name in SKIP_DIRS:
                continue
            rel = str(item.relative_to(root))
            if item.is_dir():
                yield {"path": rel + "/", "is_file": False}
                yield from _walk(root, item, max_depth, current_depth + 1)
            elif item.suffix in CODE_EXTENSIONS:
                yield {"path": rel, "is_file": True}
    except PermissionError:
        pass  # skip folders we can't read

def _read_file(path: Path) -> str | None:
    try:
        content = path.read_text(encoding="utf-8", errors="ignore")
        lines = content.splitlines()
        if len(lines) > 200:
            lines = lines[:200]
            lines.append("... [truncated at 200 lines]")
        return "\n".join(lines)
    except:
        return None

Git Interface Tool

25 minutes · Tool #2 — Claude reads git history
// The Core Mental Model
gitpython wraps the git command-line tool in Python objects. A Repo object is your entry point — it represents one git repository. From it you get commits (each with hash, message, author, date, changed files) and diffs (the actual code changes). Always read-only — never write, push, or commit through Jarvis.
pythonbackend/tools/git_interface.py
from git import Repo, InvalidGitRepositoryError
from datetime import datetime, timedelta
from pathlib import Path
import os

def read_git_history(since: str = "24h", include_diff: bool = False, file_path: str = None) -> dict:
    """
    Returns commit history for the current project.
    since: "24h", "48h", "7d", or "HEAD~3" (last 3 commits)
    Always returns a dict — never raises.
    """
    try:
        # Find the git repo (searches parent dirs too)
        project_path = get_project_path()
        repo = Repo(project_path, search_parent_directories=True)

    except InvalidGitRepositoryError:
        return {"error": "Not a git repository", "commits": []}

    try:
        # Parse the "since" parameter
        commits = _get_commits(repo, since, file_path)

        result = {
            "commits": commits,
            "total_commits": len(commits),
            "branch": repo.active_branch.name,
            "diff_summary": "",
            "diff_content": ""
        }

        if include_diff and len(repo.heads) > 0:
            diff = repo.git.diff("HEAD~1", "HEAD")
            lines = diff.splitlines()
            if len(lines) > 200:
                diff = "\n".join(lines[:200]) + f"\n... [truncated: {len(lines)} total lines]"
            result["diff_content"] = diff

        return result

    except Exception as e:
        return {"error": str(e), "commits": []}

def _get_commits(repo, since: str, file_path: str | None) -> list:
    # Handle HEAD~N format
    if since.startswith("HEAD~"):
        n = int(since.replace("HEAD~", ""))
        commits_iter = list(repo.iter_commits(max_count=n, paths=file_path))
    else:
        # Handle time-based: "24h", "7d"
        unit = since[-1]
        amount = int(since[:-1])
        delta = timedelta(hours=amount) if unit == "h" else timedelta(days=amount)
        since_dt = datetime.now() - delta
        commits_iter = list(repo.iter_commits(after=since_dt, paths=file_path))

    commits = []
    for commit in commits_iter[:20]:  # max 20 commits
        commits.append({
            "hash": commit.hexsha[:7],
            "message": commit.message.strip(),
            "author": commit.author.name,
            "date": commit.committed_datetime.strftime("%Y-%m-%d %H:%M"),
            "files_changed": list(commit.stats.files.keys())[:15]  # cap at 15
        })
    return commits
⚠
gitpython is synchronous. If you call it directly in an async function it blocks. Either wrap it: await loop.run_in_executor(None, read_git_history, since), or just call it synchronously — the git operations are fast enough (<100ms) that blocking briefly is acceptable for this use case.

Web Research Tool (Playwright)

30 minutes · Tool #3 — Claude searches the web
// The Core Mental Model
Playwright launches a real invisible browser (Chromium). It navigates to URLs, waits for the page to load, extracts text from specific HTML elements, and returns the text. Your job: give it a search query, get back relevant page content that Claude can read. The CSS selectors tell Playwright where to find the main content on each page type.
pythonbackend/tools/web_research.py
from playwright.async_api import async_playwright
import asyncio

# CSS selectors for extracting main content from different page types
# Person 5 researches these — you implement them
CONTENT_SELECTORS = [
    "article",           # blog posts, dev.to
    "main",              # most sites
    ".readme",           # GitHub README
    ".markdown-body",    # GitHub markdown
    "#readme",           # GitHub repo pages
    ".post-body",        # various blogs
    "body",              # fallback — get everything
]

async def web_research(query: str, num_sources: int = 5, purpose: str = "research_report") -> dict:
    """
    Search the web and return scraped content from top pages.
    Claude uses this content to generate the research report.
    """
    try:
        async with async_playwright() as p:
            browser = await p.chromium.launch(headless=True)
            page = await browser.new_page()

            # Step 1: Search DuckDuckGo
            search_url = f"https://duckduckgo.com/?q={query.replace(' ', '+')}&ia=web"
            await page.goto(search_url, wait_until="networkidle", timeout=15000)

            # Step 2: Extract result URLs
            links = await page.query_selector_all("a[data-testid='result-title-a']")
            urls = []
            for link in links[:8]:
                href = await link.get_attribute("href")
                if href and href.startswith("http"):
                    urls.append(href)

            # Step 3: Scrape each URL
            sources = []
            for url in urls[:num_sources]:
                content = await _scrape_page(page, url)
                if content:
                    sources.append({"url": url, "content": content})

            await browser.close()

            return {
                "query": query,
                "sources": sources,
                "source_count": len(sources)
            }

    except Exception as e:
        return {"error": str(e), "sources": [], "query": query}

async def _scrape_page(page, url: str) -> str | None:
    """Scrape one page and return its main text content."""
    try:
        await page.goto(url, wait_until="domcontentloaded", timeout=10000)

        # Try selectors in order, use first one that has content
        for selector in CONTENT_SELECTORS:
            element = await page.query_selector(selector)
            if element:
                text = await element.inner_text()
                text = text.strip()
                if len(text) > 200:  # ignore tiny content
                    # Truncate to ~3000 tokens worth of text
                    return text[:12000]

        return None

    except Exception:
        return None  # some pages block scrapers — skip them

HTML Report Generator

15 minutes · Tool #4 — saves the HTML file
// The Core Mental Model
Jinja2 is a template engine. You have an HTML file with placeholders like {{ title }}. You give Jinja2 the template and a dict of values. It replaces the placeholders and returns complete HTML. You write it to disk. Claude provides the values — you just run the template engine.
pythonbackend/tools/report_generator.py
from jinja2 import Environment, FileSystemLoader
from pathlib import Path
from datetime import datetime
import json, os

# Load templates from the templates/ folder
env = Environment(loader=FileSystemLoader("templates"))

def generate_html_report(title: str, report_type: str, sections: list, filename: str) -> dict:
    """
    Claude calls this after research is complete.
    sections: list of {heading, content, is_recommendation}
    Returns the absolute path to the saved file.
    """
    try:
        # Load the right template
        template_name = f"{report_type}_report.html"
        template = env.get_template(template_name)

        # Render with Claude's content
        html = template.render(
            title=title,
            sections=sections,
            generated_at=datetime.now().strftime("%Y-%m-%d %H:%M"),
            project_name=load_jarvis_json().get("project", "JARVIS Project")
        )

        # Save to /reports/ folder
        reports_dir = Path("reports")
        reports_dir.mkdir(exist_ok=True)

        # Filename with timestamp to avoid collisions
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        safe_name = filename.replace(" ", "_").lower()
        file_path = reports_dir / f"{safe_name}_{timestamp}.html"

        file_path.write_text(html, encoding="utf-8")

        return {
            "success": True,
            "file_path": str(file_path.resolve()),  # absolute path for Electron
            "title": title
        }

    except Exception as e:
        return {"success": False, "error": str(e)}

Memory Manager

20 minutes · Tool #5 + #6 — jarvis.json + session history
// The Core Mental Model
Two types of memory: jarvis.json (structured project facts — decisions, stack, open questions) and session summaries (what happened in each work session). Both are plain text files. The memory manager reads them at session start, updates jarvis.json when Claude calls the tool, and writes session summaries when Electron sends session_end.
pythonbackend/memory/jarvis_json.py
import json, tempfile, os
from pathlib import Path

JARVIS_JSON_PATH = Path("jarvis.json")

def load_jarvis_json() -> dict:
    """Load project memory. Returns empty dict if file doesn't exist."""
    try:
        return json.loads(JARVIS_JSON_PATH.read_text())
    except:
        return {}

def update_project_memory(field: str, operation: str, value: str) -> dict:
    """
    Claude calls this to update project memory.
    Uses atomic write (temp file → rename) to prevent corruption.
    """
    try:
        data = load_jarvis_json()

        if operation == "append":
            if field not in data:
                data[field] = []
            if isinstance(data[field], list):
                data[field].append(value)

        elif operation == "set":
            data[field] = value

        elif operation == "remove":
            if isinstance(data.get(field), list):
                data[field] = [x for x in data[field] if x != value]

        # Atomic write: write to temp file then rename
        # This prevents partial writes if the process crashes mid-write
        tmp = JARVIS_JSON_PATH.with_suffix(".json.tmp")
        tmp.write_text(json.dumps(data, indent=2))
        tmp.replace(JARVIS_JSON_PATH)  # atomic on most filesystems

        return {"success": True, "field": field, "operation": operation}

    except Exception as e:
        return {"success": False, "error": str(e)}

def load_recent_sessions(days_back: int = 3) -> str:
    """Load last N session summaries as context string."""
    sessions_dir = Path("sessions")
    if not sessions_dir.exists():
        return "No previous sessions found."

    summaries = []
    files = sorted(sessions_dir.glob("session_*.txt"), reverse=True)[:days_back]
    for f in files:
        summaries.append(f"=== {f.stem} ===\n{f.read_text()}")

    return "\n\n".join(summaries) if summaries else "No previous sessions."

def write_session_summary(summary: str):
    """Called on session_end. Saves Claude's summary to a dated file."""
    from datetime import datetime
    sessions_dir = Path("sessions")
    sessions_dir.mkdir(exist_ok=True)
    filename = f"session_{datetime.now().strftime('%Y%m%d_%H%M')}.txt"
    (sessions_dir / filename).write_text(summary)

File Watcher — Proactive Engine

30 minutes · The hardest feature — highest demo value
// The Core Mental Model
watchdog monitors a folder for file changes. When a file is modified, it calls your handler function. The debounce timer is critical — without it, one file save fires 20 events (one per character typed). With debounce: wait 5 seconds after the last event, then process once. Always routes to Ollama, never Claude API — this runs constantly in the background.
pythonbackend/context/file_watcher.py
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
import asyncio, threading

# Reference to the active WebSocket (set when Electron connects)
active_websocket = None
query_in_progress = False  # pause watcher when Claude is responding

class JarvisWatcher(FileSystemEventHandler):
    """Watches project files and triggers proactive surfaces."""

    CODE_EXTENSIONS = {'.py', '.js', '.jsx', '.ts', '.tsx'}

    def __init__(self, loop):
        self._timer = None
        self._debounce_seconds = 5
        self._loop = loop  # asyncio event loop

    def on_modified(self, event):
        if event.is_directory:
            return

        # Only care about code files
        from pathlib import Path
        ext = Path(event.src_path).suffix
        if ext not in self.CODE_EXTENSIONS:
            return

        # DEBOUNCE: cancel previous timer, start new 5-second window
        if self._timer:
            self._timer.cancel()

        self._timer = threading.Timer(
            self._debounce_seconds,
            self._process_signal,
            args=[event.src_path]
        )
        self._timer.start()

    def _process_signal(self, file_path: str):
        # Run async code from this sync thread
        if active_websocket and not query_in_progress:
            asyncio.run_coroutine_threadsafe(
                handle_file_signal(file_path),
                self._loop
            )

async def handle_file_signal(file_path: str):
    """Called after debounce. Routes to Ollama gate prompt."""
    from ai.ollama_client import run_gate_prompt
    from memory.jarvis_json import load_jarvis_json

    project_data = load_jarvis_json()
    gate_result = await run_gate_prompt(file_path, project_data)

    if gate_result.get("should_surface") and gate_result.get("confidence", 0) >= 0.7:
        from ai.ollama_client import run_surface_prompt
        bullets = await run_surface_prompt(file_path, project_data)

        if active_websocket and bullets:
            await active_websocket.send_json({
                "event": "context_surface",
                "bullets": bullets,
                "triggered_by": "file_change",
                "file_path": file_path,
                "confidence": gate_result["confidence"]
            })

async def start_file_watcher():
    """Called on FastAPI startup. Watches project source folder."""
    loop = asyncio.get_event_loop()
    handler = JarvisWatcher(loop)
    observer = Observer()

    watch_paths = load_jarvis_json().get("jarvis_config", {}).get("watch_paths", ["./src"])
    for path in watch_paths:
        from pathlib import Path
        p = Path(path)
        if p.exists():
            observer.schedule(handler, str(p), recursive=True)

    observer.start()
    print(f"File watcher started on: {watch_paths}")

    try:
        while True:
            await asyncio.sleep(1)  # keep the task alive
    finally:
        observer.stop()
        observer.join()
⚠
The watchdog runs in a thread, FastAPI runs in the asyncio event loop. They are in different worlds. You cannot call async functions directly from watchdog's thread — that's why asyncio.run_coroutine_threadsafe() is used. Without it, you get "no current event loop in thread" errors. This is the one tricky part of the file watcher.

Claude Client + Tool Use Loop

30 minutes · The brain of JARVIS
// The Core Mental Model
The Claude client is a loop. Send message. Claude responds. If Claude wants a tool: execute the tool, send result back, loop again. When Claude says stop_reason: "end_turn", the loop exits. Your job: implement the loop correctly, register all tool functions, send WebSocket status updates so Electron shows what's happening.
pythonbackend/ai/claude_client.py — complete implementation
import anthropic, json, os
from fastapi import WebSocket

# Import all tool functions
from tools.codebase_reader import read_codebase
from tools.git_interface import read_git_history
from tools.web_research import web_research

async def run(query: str, mode: str, ws: WebSocket, tool_defs: list, system_prompt: str) -> str:
    """
    Main entry point. Runs the full Claude tool_use loop.
    Sends WebSocket status events throughout so Electron can show progress.
    Returns the final text response.
    """
    messages = [{"role": "user", "content": query}]

    while True:
        await ws.send_json({"event": "status_update", "message": "Thinking...", "mode": mode})

        # Call Claude API
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1500,
            system=[{
                "type": "text",
                "text": system_prompt,
                "cache_control": {"type": "ephemeral"}  # always cache
            }],
            tools=tool_defs,
            messages=messages
        )

        # DONE — extract and return the text
        if response.stop_reason == "end_turn":
            for block in response.content:
                if block.type == "text":
                    return block.text
            return ""

        # TOOL CALL — execute the tools Claude wants
        if response.stop_reason == "tool_use":
            # 1. Append Claude's response to history
            messages.append({"role": "assistant", "content": response.content})

            # 2. Execute every tool Claude called
            tool_results = []
            for block in response.content:
                if block.type != "tool_use":
                    continue

                # Tell Electron which tool is running
                await ws.send_json({
                    "event": "tool_call_status",
                    "tool": block.name, "status": "started"
                })

                # Execute the tool
                result = await execute_tool(block.name, block.input, ws)

                # Special case: report generated — notify Electron
                if block.name == "generate_html_report" and result.get("success"):
                    await ws.send_json({
                        "event": "report_generated",
                        "file_path": result["file_path"],
                        "title": block.input.get("title", "Report")
                    })

                await ws.send_json({
                    "event": "tool_call_status",
                    "tool": block.name, "status": "completed"
                })

                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,  # MUST match — never construct manually
                    "content": json.dumps(result)
                })

            # 3. Send results back to Claude — loop continues
            messages.append({"role": "user", "content": tool_results})

async def execute_tool(name: str, inputs: dict, ws: WebSocket) -> dict:
    """Execute a tool function. Never raises — always returns a dict."""
    try:
        if name not in TOOL_REGISTRY:
            return {"error": f"Unknown tool: {name}"}

        fn = TOOL_REGISTRY[name]

        # Handle async tools (web_research uses playwright)
        import asyncio, inspect
        if inspect.iscoroutinefunction(fn):
            return await fn(**inputs)
        else:
            return fn(**inputs)  # sync tools call directly

    except Exception as e:
        return {"error": str(e), "tool": name}

Ollama Local Client

20 minutes · Free, private, always runs
// The Core Mental Model
Ollama runs a local REST API at localhost:11434. It's almost identical to the OpenAI API format. You send a prompt, it returns a response. The difference from Claude: it sometimes wraps JSON in markdown code fences. Always strip them before parsing. Used for: proactive gate (runs constantly), all calls in secure mode.
pythonbackend/ai/ollama_client.py
import httpx, json, re

OLLAMA_URL = "http://localhost:11434/api/generate"
OLLAMA_MODEL = "codellama"

async def call_ollama(prompt: str, expect_json: bool = False) -> str:
    """
    Call Ollama. If expect_json=True, strips code fences before returning.
    Returns raw text response.
    """
    try:
        async with httpx.AsyncClient(timeout=30) as client:
            response = await client.post(OLLAMA_URL, json={
                "model": OLLAMA_MODEL,
                "prompt": prompt,
                "stream": False,
                "options": {"temperature": 0}  # deterministic for gate
            })
            raw = response.json()["response"]

            if expect_json:
                return strip_json_fences(raw)
            return raw

    except Exception as e:
        if expect_json:
            return json.dumps({"should_surface": False, "confidence": 0.0, "reason": str(e)})
        return ""

def strip_json_fences(raw: str) -> str:
    """Remove markdown code fences that Ollama sometimes adds."""
    # Remove ```json ... ``` or ``` ... ```
    raw = re.sub(r'```(?:json)?\s*', '', raw)
    raw = raw.replace('```', '')
    return raw.strip()

def parse_gate_response(raw: str) -> dict:
    """Parse Ollama gate response. Returns safe default on failure."""
    try:
        # Find JSON object in the response
        match = re.search(r'\{[^{}]*\}', raw, re.DOTALL)
        if not match:
            return {"should_surface": False, "confidence": 0.0, "reason": "no JSON found"}
        return json.loads(match.group())
    except:
        return {"should_surface": False, "confidence": 0.0, "reason": "parse error"}

async def run_gate_prompt(file_path: str, project_data: dict) -> dict:
    """Run the gate — should we surface anything for this file?"""
    current_focus = project_data.get("current_focus", "general development")
    prompt = f"""Respond with ONLY a JSON object. No other text. No code blocks.

{{"should_surface": true or false, "confidence": 0.0 to 1.0, "reason": "one sentence"}}

A developer just opened: {file_path}
Project current focus: {current_focus}

JSON:"""

    raw = await call_ollama(prompt, expect_json=True)
    return parse_gate_response(raw)

async def run_surface_prompt(file_path: str, project_data: dict) -> list:
    """Generate 2-3 bullet points to surface for this file."""
    prompt = f"""Developer opened {file_path}.
Project: {project_data.get('project', 'Unknown')}
Current focus: {project_data.get('current_focus', 'general')}
Recent decisions: {project_data.get('decisions', [])}

Write 2-3 bullet points of what they should know right now about this file.
Start each bullet with •
Maximum 15 words per bullet.
Be specific to this project. No generic advice."""

    response = await call_ollama(prompt)
    # Extract bullets from response
    bullets = [line.strip().lstrip("•-").strip()
               for line in response.splitlines()
               if line.strip().startswith(("•", "-"))]
    return bullets[:3]

import os
from memory.jarvis_json import load_jarvis_json

def get_model(task_type: str = "general") -> str:
    """
    Returns the model string for a given task type.
    Reads AI_MODE from jarvis.json first, falls back to env var.
    """
    config = load_jarvis_json().get("jarvis_config", {})
    ai_mode = config.get("ai_mode", os.getenv("AI_MODE", "local"))

    # Secure mode: everything goes to Ollama
    if ai_mode == "local":
        return "ollama"

    # Cloud mode: route by task quality requirement
    sonnet_tasks = {"research_report", "excel_analysis", "error_diagnosis"}
    haiku_tasks  = {"git_summary", "commit_message", "session_summary",
                    "pr_description", "quick_qa", "project_setup"}
    ollama_tasks = {"proactive_gate", "relevance_scoring"}

    if task_type in sonnet_tasks:  return "claude-sonnet-4-20250514"
    if task_type in haiku_tasks:   return "claude-haiku-4-5-20251001"
    if task_type in ollama_tasks:  return "ollama"
    return "claude-haiku-4-5-20251001"  # default to haiku

Error Handling Rules

10 minutes · The rules that prevent silent failures

Rule 1: Tools never raise exceptions — they return error dicts.

If a tool crashes and raises, the entire tool_use loop breaks and Jarvis goes silent. Wrap every tool function body in try/except. Return {"error": str(e)} instead of raising. Claude reads the error and tells the user what happened.

❌ Will break the loop silently
def read_git_history(since):
  repo = Repo(".")
  # raises if not a git repo
  return commits
✓ Fails gracefully
def read_git_history(since):
  try:
    repo = Repo(".")
    return commits
  except Exception as e:
    return {"error": str(e)}

Rule 2: Rate limit errors must retry, not crash.

pythonRate limit retry wrapper — add this to claude_client.py
import time, anthropic

def call_with_retry(fn, max_retries=4):
    """Wraps any Claude API call with exponential backoff on rate limits."""
    for attempt in range(max_retries):
        try:
            return fn()
        except anthropic.RateLimitError:
            wait = 2 ** attempt   # 1s, 2s, 4s, 8s
            print(f"Rate limited. Waiting {wait}s...")
            time.sleep(wait)
        except anthropic.APIConnectionError:
            # API is down — fall back to Ollama
            return None  # caller handles None → switches to Ollama
    raise Exception("Max retries exceeded")

Rule 3: Always show sample tool output to AI Lead before writing prompts.

Before considering a tool "done," run it on the actual project and paste the output in the shared doc. AI Lead writes prompts against exact field names. If you rename a field after they've written prompts, the prompts silently break.

Rule 4: The tool_use_id must always come directly from block.id.

Never construct it. Never copy it by hand. Always: tool_result["tool_use_id"] = block.id. A mismatch causes an immediate API 400 error that is hard to diagnose.
