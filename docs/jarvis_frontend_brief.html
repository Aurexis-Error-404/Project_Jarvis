Section 01

Your job in one paragraph

You own everything the developer sees. That's three things: the hotkey overlay (Ctrl+Space), the proactive surface card that appears automatically when Jarvis detects relevant context, and the SECURE MODE badge that shows which AI engine is active. You receive everything over WebSocket from the FastAPI backend — you never call the Claude API directly.

Component 1
Hotkey Overlay
Ctrl+Space opens a floating panel. Input box, streaming response, history. Primary interface. Must open in <100ms.
Component 2
Surface Card
Auto-appears bottom-right when proactive engine fires. 2-3 bullet points. Dismissable. Never interrupts typing.
Component 3
Secure Mode Badge
Toggle in overlay header. Shows LOCAL / CLOUD. Toggle sends mode change to backend over WebSocket. Visible in demo.
What you don't own: The Claude API, tool execution, file watching, Ollama. Those all live in the Python backend. You consume events, you don't produce AI outputs.

Section 03

WebSocket contract

Endpoint: ws://localhost:8765 — connect once on app start, reconnect on disconnect with 2s backoff. All messages are JSON.

Events you RECEIVE from backend

jarvis_stream_chunk
Streaming text from Claude. Append event.text to streamingText. When event.done === true, stop streaming indicator.
jarvis_surface
Proactive engine fired. Show surface card with event.bullets[] and event.file. Auto-dismiss after 8s.
jarvis_response
Non-streamed response (used during tool_use loop). Replace full response area. Not streaming.
jarvis_mode_ack
Backend confirmed mode switch. Update badge. Payload: {mode: 'local' | 'cloud'}
jarvis_error
Backend error. Show inline in overlay — never crash the UI. Payload: {message: string}

Events you SEND to backend

user_query
User submitted a query from the overlay. Payload: {text: string, mode: 'local'|'cloud'}
mode_change
User toggled secure mode. Payload: {mode: 'local' | 'cloud'}. Wait for jarvis_mode_ack before updating badge.
surface_dismissed
User dismissed the surface card. Payload: {file: string}. Backend uses this for cooldown tracking.

WebSocket hook — copy this exactly

// hooks/useWebSocket.js
import { useEffect, useRef } from 'react';

export function useWebSocket(dispatch) {
  const ws = useRef(null);
  const reconnectTimer = useRef(null);

  const connect = () => {
    ws.current = new WebSocket('ws://localhost:8765');

    ws.current.onopen = () => {
      dispatch({ type: 'WS_CONNECTED' });
    };

    ws.current.onmessage = (e) => {
      const event = JSON.parse(e.data);
      switch (event.event) {
        case 'jarvis_stream_chunk':
          if (event.done) {
            dispatch({ type: 'STREAM_DONE' });
          } else {
            dispatch({ type: 'STREAM_CHUNK', text: event.text });
          }
          break;
        case 'jarvis_surface':
          dispatch({ type: 'SHOW_SURFACE', bullets: event.bullets, file: event.file });
          break;
        case 'jarvis_mode_ack':
          dispatch({ type: 'MODE_CONFIRMED', mode: event.mode });
          break;
        case 'jarvis_error':
          dispatch({ type: 'SET_ERROR', message: event.message });
          break;
      }
    };

    ws.current.onclose = () => {
      dispatch({ type: 'WS_DISCONNECTED' });
      // Reconnect after 2s
      reconnectTimer.current = setTimeout(connect, 2000);
    };
  };

  useEffect(() => {
    connect();
    return () => {
      ws.current?.close();
      clearTimeout(reconnectTimer.current);
    };
  }, []);

  return ws;
}React
Section 04

Electron IPC bridge

Critical: The surface card event comes from the Python backend → WebSocket → Electron main process. It will NOT reach React automatically. You must forward it via IPC or it silently disappears.

main.js — global hotkey + IPC forwarding

// main.js (Electron main process)
const { app, BrowserWindow, globalShortcut, ipcMain } = require('electron');
const WebSocket = require('ws');

let mainWindow;
let ws;

app.on('ready', () => {
  mainWindow = new BrowserWindow({
    width: 680,
    height: 480,
    frame: false,           // frameless window for overlay feel
    alwaysOnTop: true,      // floats above IDE
    show: false,             // hidden until Ctrl+Space
    webPreferences: {
      contextIsolation: true,
      preload: __dirname + '/preload.js'
    }
  });

  // Ctrl+Space toggles the overlay
  globalShortcut.register('Control+Space', () => {
    if (mainWindow.isVisible()) {
      mainWindow.hide();
    } else {
      mainWindow.show();
      mainWindow.focus();
    }
  });

  // Forward backend surface events to React
  // This is the line most people forget
  ws = new WebSocket('ws://localhost:8765');
  ws.on('message', (data) => {
    const event = JSON.parse(data);
    if (event.event === 'jarvis_surface') {
      mainWindow.webContents.send('context_surface', event);
    }
  });
});JS

preload.js — expose IPC to React safely

// preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('jarvis', {
  // React calls this to listen for surface events from main
  onSurface: (callback) =>
    ipcRenderer.on('context_surface', (_e, data) => callback(data)),

  // React calls this to remove listener on unmount
  offSurface: (callback) =>
    ipcRenderer.removeListener('context_surface', callback),
});JS

In React — listen for surface events from main process

// In your App or SurfaceCard component
useEffect(() => {
  const handler = (event) => {
    dispatch({
      type: 'SHOW_SURFACE',
      bullets: event.bullets,
      file: event.file
    });
  };

  window.jarvis.onSurface(handler);
  return () => window.jarvis.offSurface(handler);
}, []);React
Section 05

What good output looks like

BAD — do not ship

✗ Overlay takes 500ms+ to open after Ctrl+Space
✗ Surface card appears in the middle of the screen — interrupts typing
✗ SECURE MODE badge is hidden — not visible in demo without hunting for it
✗ Streaming text appears all at once (buffering somewhere)
✗ WebSocket disconnects and never reconnects — needs app restart

GOOD — ship this

✓ Overlay opens in under 100ms — feels instant
✓ Surface card slides in from bottom-right, stays out of the way
✓ SECURE MODE badge visible immediately, toggles live during demo
✓ Text streams word-by-word with a blinking cursor
✓ Auto-reconnects silently if backend restarts

Demo-critical requirement — the toggle

The judge will ask to see SECURE MODE toggled live. The badge must visibly change between LOCAL and CLOUD and the backend must confirm the switch before the badge updates. Don't fake this — it needs to actually change the mode over WebSocket.
