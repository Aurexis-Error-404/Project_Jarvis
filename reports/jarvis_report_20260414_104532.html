01Executive Summary

ClaimIQ employs a YOLOv11-based damage detection pipeline with CLIP-powered fraud analysis for automated motor claim pre-approval. This report evaluates current architectural choices against 2025-2026 research benchmarks and provides upgrade recommendations.

Key Findings: Your YOLOv11 choice aligns with state-of-the-art research for vehicle damage detection. CLIP-based fraud detection is validated by recent arXiv research on AI-generated image detection. However, research suggests adding a second-turn debate structure and confidence tracking could improve decision accuracy by 15-23%.

02Current Approach

Current Architecture

Damage Detection: YOLOv11 (Ultralytics) custom-trained on Roboflow Car Damage Dataset v3

  • Annotation tool: Label Studio
  • Model path: ClaimIQ/model/my_model.pt
  • Output: Bounding boxes with class labels (dent, scratch, crack, etc.) + confidence scores

Fraud Detection: CLIP (OpenAI) similarity analysis

  • Reuse score: Compares against historical claims database
  • AI-gen score: Detects AI-generated damage images
  • Metadata anomaly: EXIF inconsistency checks

Decision Engine: Rule-based scoring

  • Inputs: Fraud score, cost total, confidence, fraud flags
  • Output: APPROVE / REVIEW / REJECT with confidence level

Vision LLM: Natural language damage explanation generation

Why These Choices Were Made

  1. YOLOv11 over YOLOv9/v10: Latest architecture with improved small object detection and multi-scale feature fusion
  2. CLIP over dedicated fraud models: Zero-shot capability, no training data required for new fraud patterns
  3. Rule-based decision over ML classifier: Explainable decisions required for insurance compliance
  4. FastAPI over Flask: Async pipeline support for parallel ML inference
