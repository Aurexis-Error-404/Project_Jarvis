01Executive Summary

ClaimIQ is an AI-powered motor claim pre-approval system that automates damage assessment, cost estimation, and fraud detection for insurance claims. The system processes claim images through a custom YOLOv11 damage detection model, generates natural language explanations via Vision LLM, estimates repair costs, analyzes fraud risk using CLIP similarity, and renders instant approve/review/reject decisions.

Target Users: Insurance companies, claim adjusters, policyholders seeking instant pre-approval

Key Value: Reduces claim processing time from days to seconds with explainable AI decisions and annotated damage overlays.

04ML Pipeline Details

Custom YOLOv11 Model

  • Training Data: Roboflow Universe – Car Damage Dataset v3
  • Annotation Tool: Label Studio
  • Model Path: ClaimIQ/model/my_model.pt
  • Environment Config:

  YOLO_MODEL_PATH=../model/my_model.pt


  YOLO_WEIGHTS_DIR=../model/train/weights

Detection Output

  • Returns bounding boxes with class labels (dent, scratch, crack, etc.)
  • Confidence scores per zone
  • Generates annotated overlay images uploaded to Supabase
  • Frontend displays YOLO-processed images after analysis

Fraud Detection Signals

  1. Reuse Score — CLIP similarity to historical claims
  2. AI-Gen Score — Detection of AI-generated damage images
  3. Metadata Anomaly — EXIF inconsistencies
  4. Frequency Analysis — Repeat claims from same user/vehicle

Decision Engine

  • APPROVE — Low fraud, moderate cost, high confidence
  • REVIEW — Medium fraud flags or low confidence detections
  • REJECT — High fraud score or critical inconsistencies

09End-to-End Test Flow

  1. Start Backend:

   cd ClaimIQ/backend


   .\.venv311\Scripts\python.exe -m uvicorn app.main:app --reload

Verify: http://127.0.0.1:8000/health

  1. Start Frontend:

   cd ClaimIQ/frontend


   npm run dev

Verify: http://localhost:3000

  1. Test Flow:
  • Register new account
  • Create claim, upload 1-3 damage images
  • Open claim detail page
  • Click Run AI Analysis
  • Verify:
  • Status changes to "processed"
  • Damage zones appear with confidence scores
  • Photos tab shows YOLO-annotated images
  • Cost breakdown displays itemized totals
  • Fraud score and flags populate
  • Decision (APPROVE/REVIEW/REJECT) shows with confidence
  1. Troubleshooting:
  • If API calls fail: check backend running on 8000, frontend on 3000
  • If YOLO fails: verify model/my_model.pt exists
  • If CLIP warnings appear: install CLIP or ignore (non-blocking)
