01Executive Summary

AGENT_IITH is a multi-agent stock analysis system built for Forge Inspira 2026 hackathon under the theme 'Autonomous System That Takes Action'. The system uses adversarial debate between Bull, Bear, and Risk agents with a Mediator synthesizing final recommendations. Built with Node.js/Express backend, React 18 + Vite frontend, Supabase database, and Qwen2.5-72B-Instruct via Featherless.ai.

03Agent Roles & Prompts

| Agent | Role | Output Bias | Key Focus | |-------|------|-------------|----------| | Bull | Identify buy signals | Optimistic | Revenue growth, market share, positive catalysts | | Bear | Identify sell signals | Pessimistic | Risks, competition, downside scenarios | | Risk | Flag dangers | Conservative | Volatility, regulatory issues, red flags | | Mediator | Synthesize all three | Neutral | Weight evidence, resolve conflicts, assign confidence |

Output Format: Each agent returns structured JSON with:

  • recommendation: BUY/SELL/HOLD
  • confidence: 0-100 score
  • reasoning: Array of specific data points with sources
  • conditional_triggers: Events that would change the recommendation

06Key Differentiators

  1. Reasoning Transparency — Shows full decision trail, not just final recommendation
  2. Adversarial Debate — Conflicting agents surface both bullish and bearish cases
  3. Conflict Resolution — Mediator explicitly resolves disagreements with weighted evidence
  4. Conditional Triggers — Each recommendation includes events that would change the stance
  5. Real-time Streaming — Results appear as agents complete, not after all finish

Hackathon Alignment: Directly addresses 'Autonomous System That Takes Action' theme — system independently gathers data, analyzes through multiple perspectives, and produces actionable recommendations without human intervention.

08Demo Strategy

Recommended Demo Flow

  1. Start with AAPL — Already tested, stable output
  2. Show reasoning trail — Expand Bull/Bear cards to display specific data points
  3. Highlight conflict — Point out where Bull and Bear disagree, show Mediator resolution
  4. Run 2-3 more tickers — Demonstrate system works across different sectors
  5. Show Supabase data — Quick peek at persistence layer

Fallback Plan

  • Pre-recorded analysis videos if live scraping fails
  • Cached results for 3 tickers if API rate limits hit
  • Single-agent mode if multi-agent orchestration has issues

Judge Talking Points

  • Emphasize autonomous data gathering (no manual input beyond ticker)
  • Highlight adversarial approach vs. single-model summaries
  • Show confidence scores and how they correlate with agent agreement
